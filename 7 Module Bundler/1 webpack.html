모듈 번들러: 웹팩/Parcel 여러개의 파일을 하나의 파일로 묶어주는 개발 자동화 도구인 webpack webpack는 파일을 묶는 과정에서 여러가지 작업들이 실행되도록 작업 계획을 세워주기도 합니다. sass, typescript, jsx와 같은 언어들을 네이티브 언어로 컴파일 해주거나, 개발 중에 파일이 수정 되었을 때 자동으로 브라우저의 내용을 리로드 해주는 등의 작업도 할 수 있습니다. 모듈 번들러는 빌드 과정의 문제를 해결해주는데, 라이브러리의 js 파일과 자기가 짠 js 코드를 하나의 js로 묶어주는(bundling) 역할을 한다. 이건 클라이언트에서 js 파일을 하나만 불러와도 되니까 시간적인 이점도 있지만, 대부분의 브라우저에서 자바스크립트 모듈을 받아들이지 않기 때문에 require 처럼 브라우저에서 알아듣지 못하는 js 문법을 실제로 필요한 파일의 콘텐츠로 바꿔주는 역할도 같이 한다는 이점도 있다. 모듈 번들러의 결과로는 require 없는 단일 번들 js 파일이 나오게 된다. 뿐만 아니라 webpack의 설명을 보면 같은 유형의 파일(css, js 등)은 하나로 묶을 수 있다. 모듈 번들러는 컴파일 시간에 번들 파일 생성을 위해 실행된다. 그러면 브라우저에서 그 결과로 나온 번들 파일을 로드하는 것이다. 개발자 콘솔에서 네트워크 탭을 보면 1개 파일만 로드된 것을 확인할 수 있다. 명령어: webpack 설치: npm install -D webpack webpack-cli webpack으로 번들링: npx webpack --entry ./source.index.js --output .public.index_bundle.js
