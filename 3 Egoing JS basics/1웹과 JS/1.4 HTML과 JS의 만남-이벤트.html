<이벤트 : 브라우저에서 다양한 이유로 발생하는 사건들> (Javascript와 사용자의 상호작용, 이벤트) --> 이러한 이벤트에 대처하는 프로그램 : 이벤트 핸들러(주로 함수 하나로 구성됨) 브라우저에서 발생하는 이벤트는 크게 4가지 * Form 이벤트 : HTML 문서의 폼 엘리먼트에 변화가 생겼거나 제출 버튼 등이 눌렸을 때 발생하는 이벤트 * Window 이벤트 : 페이지가 모두 로드되었을 때 발생하는 이벤트로 onload event가 대표적 * MouseEvent : 사용자가 마우스를 조작했을 때 발생하는 이벤트 * KeyboardEvent : 사용자가 키보드를 조작했을 때 발생하는 이벤트 >이벤트 종류: --> click 이벤트는 = 마우스 이벤트(마우스로 클릭했을때 발생) --> keydown 이벤트는 = 키보드 이벤트(키가 눌렸을때 발생) --> change 이벤트는 = 폼 이벤트(form 엘리먼트의 내용이 변경되었을 때 발생) 예제:

HTML 태그 속성에 이벤트 핸들러 추가하기

HTML 태그 안에서 onclick 속성은 javascript 코드를 가지게 됨. 그리고 onclick이 포함된 태그가 클릭되었을 때 이 javascript 코드에 따라서 웹 브라우저가 동작됨. 즉, 위의 코드에서 웹 브라우저는 console.log('clicked') 라는 코드를 기억하고 있다가, 사용자가 클릭하면 이를 실행해주는 것. 이렇게 웹 브라우저에서 일어나는 유용한 사건을 이벤트(Event)라고 함. >예제 2: alert('hi') 위에서는 그냥 경고창이지만 아래는 경고창을 열어주는 버튼을 생성 *** 매우 많은 이벤트들이 존재하며 코딩을 하다가 필요한 이벤트들은 구글링을 통해서 쉽게 찾아 볼 수 있음. 예를 들어 키보드를 누르는 이벤트를 알아보고 싶다면, "javascript keydown event"라고 검색하면 onkeydown 이벤트를 찾아볼 수 있음
